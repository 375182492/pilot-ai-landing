WHY PILOT AI

不只是一个聊天框，而是可编排的智能体运行平台

AGUI 开放协议、A2UI 自定义渲染、多层次记忆与全链路可观测，让 Agent 的"思考过程"与"运行状态"同样可被看见。

🌐

AGUI / A2UI 开放协议

遵循 AG-UI 协议规范，后端通过 SSE 推送 12 类标准事件（RUN_STARTED → STEP → TEXT_DELTA / TOOL_CALL → RUN_FINISHED），前端按事件动态驱动可视化组件渲染。A2UI 扩展通过 CUSTOM_UI_EVENT 携带图表、表格、表单、对比矩阵等业务数据。

🌊

SSE 流式 + 思考可见

TEXT_DELTA 逐 token 打字机渲染，THINKING_DELTA 实时展示推理链路（Chain-of-Thought）。Nginx 关闭 proxy_buffering，断线指数退避自动重连，后端保留 run 状态 30 秒。

🤖

AgentScope 2.0 引擎

以 AgentScope 2.0 Java SDK 为核心运行框架，Pipeline 编排 LLM 调用与工具调用，监听各步骤回调发射事件，支持多轮迭代与人工介入（Human-in-the-Loop）。

📊

全链路可观测

每次对话独立 run 记录，Trace Span 精确到 LLM_CALL / TOOL_CALL / PIPELINE_STEP，记录输入输出、Token 消耗、耗时与状态，前端 TraceViewer 甘特图可视化复盘。

AGENT CAPABILITIES

Agent 能力：会思考、会用工具、会等人

每一个智能体都具备完整的推理—行动—反馈闭环，并支持人工介入与多轮工具编排。

🔁

Pipeline 推理循环

系统提示词 + 上下文 + 工具声明驱动 LLM 决策，最大迭代次数（默认 10）内自主调用工具、整合结果、形成最终回答。

🧰

多类型工具调用

内置工具 / MCP 工具 / HTTP API 工具统一注册。TOOL_CALL_START → ARGS_DELTA → TOOL_RESULT 全过程前端可见，超时可控。

🧠

长短记忆协同

短期记忆承载会话上下文，长期记忆沉淀用户偏好与事实；AGENT_CONTROL 模式让 Agent 自主决定记忆写入。

🙋

Human-in-the-Loop

FORM_REQUEST / CONFIRM_REQUEST / SELECTION_REQUEST 暂停 Agent 执行，等待用户填写表单或确认操作后恢复，挂起/恢复机制完备。

🔎

RAG 知识增强

绑定知识库后自动检索召回，KNOWLEDGE_CHUNKS 事件展示引用块与相关度分数，回答可溯源。

📈

运行可追溯

每次 run 落库 Agent 运行记录与 Trace Span，运行/空闲/错误状态实时徽章，支持消息重放。

EXTENSIBILITY

扩展能力：协议即边界，可插拔生长

平台以"开放协议 + 插件化工具 + 自定义 UI"为扩展哲学，无需改动核心即可接入新能力。

🔌

MCP 工具生态

原生支持 MCP（Model Context Protocol）工具接入，一行配置把外部能力（数据库、API、文件系统）变成 Agent 可调用的 Skill。

🎨

A2UI 自定义渲染

通过 CUSTOM_UI_EVENT 携带任意业务数据，前端组件库按 eventName 动态路由 —— 折线图、对比矩阵、地图热力、动态表单皆可流式渲染。

🧱

内置工具与代码工具

支持 builtin / mcp / http / code 四类工具；开发者可编写自定义工具并注册到 Skills 服务，按需绑定给任意智能体。

🗄️

知识库与模型可替换

内置向量库（Milvus/Chroma/SQLite 可切换）与 Dify 外部知识库并行支持；模型供应商即插即用，开发环境可本地 Ollama

🚀

AguiSseHandler.java — 事件映射核心
// 将 AgentScope 事件映射为 AGUI 标准事件
pipeline.onThinkingDelta(d ->
    emit(emitter, AguiEventType.THINKING_DELTA, Map.of("delta", d)));
pipeline.onTextDelta(d ->
    emit(emitter, AguiEventType.TEXT_DELTA, Map.of("delta", d)));
pipeline.onToolCallStart(tc ->
    emit(emitter, AguiEventType.TOOL_CALL_START,
        Map.of("toolCallId", tc.getId(),
              "toolName", tc.getName())));
RunResult result = pipeline.run(userMsg);
emit(emitter, AguiEventType.RUN_FINISHED,
    Map.of("totalTokens", result.getTokens()));
